README structure

Your README should have:

1. Project overview

What the system does.

2. Features

List of functionality.

3. Architecture

Simple diagram:

Binance API → Backend → DB → Backtester → Frontend

4. Setup instructions

How to install and run.

5. Sample output

Screenshots and key metrics.

6. Design decisions

Why you chose FastAPI, SQLite, MA crossover.

7. Limitations

Example:

long-only

daily candles only

no slippage model
